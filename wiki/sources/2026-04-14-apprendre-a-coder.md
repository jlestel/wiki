# Apprendre à coder — l'IA ne remplace pas la compétence

## Résumé

Thread de [[brivael]] contre la narrative « le code est mort ». Thèse : l'IA est un levier qui amplifie une compétence existante, pas un remplacement. Il faut maîtriser les fondamentaux (data structures, algorithmes, systèmes) pour construire avec l'IA. Computer science est un art, pas une science.

## Affirmations clés

- « Le code est mort, plus besoin d'apprendre à coder » = la pire narrative actuelle
- Computer science est un art, comme la peinture ou la musique — il faut maîtriser ses gammes
- **L'IA est un levier** : amplifie une force existante. Force à zéro × 1000 = zéro
- 3 piliers à maîtriser : voir [[fondamentaux-cs]]
- C'est la pensée en couches qui sépare quelqu'un qui « utilise l'IA » de quelqu'un qui construit avec
- L'IA ne remplace pas la compétence, elle l'amplifie

## Les 3 piliers

1. **Data structures** — array, hashmap, arbre, graphe. Le vocabulaire pour décrire le monde en code
2. **Algorithmes** — trier, chercher, parcourir, optimiser. La grammaire qui donne du sens
3. **Systèmes** — penser à l'échelle. Orchestrer des technologies qui wrappent data structures et algorithmes

## Livres recommandés

- *Grokking Algorithms* (Bhargava) — point d'entrée, illustré, accessible
- *Algorithms, 4th Edition* (Sedgewick & Wayne) — référence académique
- *System Design Interview Vol. 2* (Alex Xu) — systèmes réels (load balancers, caches, message queues)
- *Cracking the Coding Interview* (McDowell) — pensée structurée déguisée en prep d'interview
