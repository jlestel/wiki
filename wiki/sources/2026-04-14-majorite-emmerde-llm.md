# La majorité m'emmerde — LLMs et biais de la majorité

## Résumé

Billet d'un adminsys « vieille école » frustré par les LLMs qui produisent du « good-enough » calibré sur les 90% d'utilisateurs mainstream. Son architecture réseau classique (une machine = une mission) est une « curiosité » pour ChatGPT. Il doit blinder ses instructions (AGENTS.md) juste pour contrer les biais statistiques, ce qui brûle ses tokens.

## Points clés

- LLMs = connaissance pondérée/moyennée, pas brute. Le prochain token est « supposé approprié » mais pas « juste »
- Données d'entraînement : le contenu « good-enough » surreprésenté noie le contenu expert
- 90% des utilisateurs veulent du good-enough → les 10% restants doivent se battre contre les biais
- Le temps passé à écrire des instructions correctrices est du temps perdu sur la tâche réelle
- Les pratiques minoritaires (adminsys classique, Merise, UML, pré-Agile) risquent l'extinction dans les modèles
- « Apprendre à coder, ça ne veut pas dire suivre une formation de 15 jours sur JavaScript » → rejoint [[fondamentaux-cs]]
