# Saying Goodbye to Agile

## Résumé

Billet de [[lewis-campbell]] qui enterre Agile. Thèse : Agile s'est imposé comme négation de Waterfall, mais les vertus qu'il revendiquait — prototypage, itération, implication client — étaient déjà prescrites par [[winston-royce]] en 1970. L'émergence des LLM remet les spécifications au centre : « Spec-Driven Development ». La documentation *est* le design — idée vieille de 50 ans, redevenue centrale.

## Affirmations clés

- Agile n'a jamais été défini clairement : ses défenseurs disent « ce n'est pas le vrai Agile » dès qu'on questionne, et le manifeste lui-même est vague ou commercialement ingérable (« welcome changing requirements, even late in development »)
- Agile était défini négativement : « tout ce qui n'est pas Waterfall »
- Royce 1970 (« Managing the Development of Large Software Systems ») recommandait déjà : design d'abord, prototype, implication formelle du client
- Bell & Thayer 1976 (qui forge le terme « Waterfall ») montrait déjà empiriquement que les besoins évoluent à mesure du design
- L'itération n'était donc pas une innovation Agile mais un acquis des années 1970
- Les LLM, comme beaucoup de développeurs, gèrent mal l'ambiguïté : spécifier produit du code correct
- Agile : « working software over comprehensive documentation ». Spec-Driven : « comprehensive documentation creates working software »
- Royce : « documentation, specification, design désignent une même chose avant le code. Mauvaise documentation = mauvais design. »
- Agile a été « une solution vague à un problème déjà résolu par des ingénieurs sérieux un demi-siècle plus tôt »
