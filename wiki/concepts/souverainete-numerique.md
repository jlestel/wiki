# Souveraineté numérique

Capacité d'un État à maîtriser son infrastructure numérique sans dépendre des GAFAM.

## Trois piliers (analyse Adrien.D)

### 1. Sécurité (approche [[anssi]])

- La sécurité par l'obscurité ne marche pas
- Windows = boîte noire, télémétrie non auditable, soumise au Cloud Act
- Code libre = code auditable

### 2. Interopérabilité (leçon allemande)

- Le vrai piège Microsoft n'est pas Windows mais l'écosystème fermé (formats propriétaires)
- Formats ouverts (ODF) = capacité de changer de fournisseur
- La France impose Word = client forcé de Microsoft

### 3. Cloud souverain ≠ localisé

- Un « cloud souverain » qui n'est qu'un Microsoft hébergé en France est une illusion
- Souveraineté = piles logicielles libres, code maîtrisé, agents internes

## Annonces France (avril 2026)

- [[dinum]] sort de Windows → Linux
- CNAM : 80 000 agents migrent vers socle open source
- Plateforme données de santé sur solution de confiance d'ici fin 2026
- Coalitions public/privé pour créer des communs numériques
- [[2026-04-12-france-linux-opensource]]

## Contexte européen

- Danemark 2025 : migration Linux + LibreOffice
- Allemagne : format ODF imposé
- France : premières « rencontres industrielles du numérique » en juin 2026
